## 0. 포지셔닝: 핵심은 엔진

**핵심 코어는 엔진이다.** — 언리얼 엔진처럼, 4D 시뮬레이션 런타임이 중심.  
동시에 **엔진 + God View를 합치면 제품 그 자체로도 바로 쓸 수 있다.**

| 구분 | 설명 |
|------|------|
| **핵심** | 4D 좌표, 5대 규칙, 시간 흐름, 3계층 감정·생각 생성 — 이 **엔진**이 전부 |
| **제품 사용** | 엔진 + God View = 완성된 시나리오 탐색 도구. 이 형태로 그대로 사용 가능 |
| **확장** | 엔진 API를 다른 UI·제품에 임베드하거나, 정책·금융·교육용 제품으로 재구성 가능 |

→ 제품으로 써도 되고, 엔진만 쓰려 해도 된다. **중심은 항상 엔진**.

## 2. 핵심 컨셉

### 2.1 세계관

- **에이전트** = 하나의 유기체 세포
- **세계** = 4차원 시공간 (x, y, z, t)
- **시간(t)** = 실제로 흐름. 세포들이 자연스럽게 성장·분열·사멸·융합·돌연변이를 겪음
- **감정·생각·세계관** = 3계층 벡터 (Emotion 단기 → Thought 중기 → Worldview 장기). 세포가 경험을 통해 스스로 만들어 냄

### 2.2 기존 모델과의 차이

| 구분 | 기존 멀티에이전트 | Organic4D |
|------|-------------------|-----------|
| 시간 | 주로 단일 실행 또는 짧은 시뮬레이션 | t가 실제로 흐르며 장기 진화 |
| 개체 수 | 고정 또는 외부 제어 | 분열·사멸로 **자동 증감** |
| 감정·생각·세계관 | 외부 주입 또는 고정 | 3계층 메커니즘 기반 **자생** |
| 목표 | 단기 의사결정 | 5~10년 **emergent 현상** 관찰 |

## 3. 핵심 목표 (Goal Setting)

> **우선순위 변경**: 목표의 방향은 그대로 두고, **효용의 우선순위**만 조정. 복잡계 이해·시나리오 탐색·의사결정 지원을 1순위로 둠.

### 3.1 목표 전체 (방향 유지)

| 목표 | 설명 |
|------|------|
| **복잡계 이해** | 정책·금융·기후·사회 같은 복잡계를 시각화·추상화해 구조와 동역학을 직관적으로 파악 |
| **시나리오 탐색** | What-if 질문 ("이 정책이 통과되면?", "이 사건이 발생하면?")에 대한 가능성 공간 탐색 |
| **의사결정 지원** | 단일 "정답"이 아닌 여러 시나리오 비교·인사이트 제공으로 의사결정자의 판단을 보조 |
| **장기 예측·흐름 탐색** | 5~10년 단위로 복잡계 변화 패턴·추세를 현실적으로 탐색 (정성적 패턴 + 장기 시뮬레이션) |
| **Emergent 현상 관찰** | 인간 개입 없이 이념 집단 형성, 혁명, 멸종, 새 종 출현을 자동 관찰 |
| **진화 수준 구현** | 기존 멀티에이전트가 못 하는 "살아있는 생명체 수준의 진화" 구현 |

### 3.2 우선순위 (효용 강조 순)

1. **복잡계 이해** · **시나리오 탐색** · **의사결정 지원** — 엔진이 이 역량을 지원해야 함 (제품 포지셔닝 1순위)
2. **장기 예측·흐름 탐색** · **Emergent 현상 관찰** · **진화 수준 구현** — 핵심 목표 그대로 유지

## 4. 생물학적 5대 규칙

| 규칙 | 조건 | 결과 |
|------|------|------|
| **성장** | 영양분 흡수 | 에너지 증가 |
| **분열** | 에너지 > 임계치 | 1개 세포 → 2개 (유전자 약간 변이) |
| **사멸** | 에너지 = 0 | 죽음 + 영양분을 주변 세포에 분배 |
| **융합** | 가까운 거리 + **Thought 유사도 0.7+** (70%) + Worldview 호환 | 하나의 강한 세포로 합체 |
| **돌연변이** | 분열/융합 시 | 유전자·감정·생각·세계관 벡터가 일부 변이 |

## 5. 세포(에이전트) 상태

```
세포 = {
  위치: (x, y, z, t)
  에너지: number
  유전자_벡터: vector
  메모리: [장기 이벤트들]
  // 3계층 감정·생각·세계관
  emotion_vec: vector   // 8~128차원, 단기 기분
  thought_vec: vector   // 256차원, 중기 전략
  worldview_vec: vector // 384차원, 장기 신념
}
```

- **유전자 벡터**: 분열·융합 시 자식/합체체에 전달·변이
- **메모리**: 경험한 이벤트(뉴스·정책·상호작용)의 장기 저장
- **3계층 벡터**: 아래 6장 참조

## 6. 3계층 감정·생각 생성 메커니즘

> 기존 "사상"이 너무 추상적이어서 도입. 단기·중기·장기로 계층화해 구현·시각화·비용 관리 용이.

### 6.1 Emotion Vector (감정 벡터) — 단기 '기분' 레이어

| 항목 | 내용 |
|------|------|
| **역할** | 지금 이 순간 세포가 느끼는 감정. 에너지·상호작용·God View 주입에 **즉시** 반응 |
| **차원** | 8차원 (joy, anger, fear, calm 등 기본 감정 가중치) 또는 128차원 |
| **업데이트** | 매 t마다 (또는 이벤트 발생 시 1초 이내). **LLM 불필요**, 규칙 기반 |
| **시각화** | 세포 **색상** (빨강=분노, 파랑=안정) + **크기** (감정 강도) 실시간 변화 |

**예시**: t=300에 "화성 자원 부족 정책" 주입 → 전체 세포 감정이 '불안 0.9'로 폭등 → 화면 전체 붉게, 사멸률 3배. SF 영화 사회 패닉 장면.

### 6.2 Thought Vector (생각 벡터) — 중기 '전략' 레이어

| 항목 | 내용 |
|------|------|
| **역할** | 현재 세포가 하는 구체적 생각 ("자원 부족 → 주변과 합쳐야겠다") |
| **차원** | 256차원 |
| **업데이트** | 10~50 t마다 LLM 한 번 (Ollama 3B, 3초 이내) |
| **용도** | **융합·분열 조건의 70%** — cosine similarity 0.7 이상이면 자동 협력 |

**예시**: 자원 부족 지속 → "협력하자" 생각 퍼짐 → 클러스터가 대형 세포로 진화. "이 집단이 생존 전략을 바꿨구나"가 한눈에 보임.

### 6.3 Worldview Vector (세계관 벡터) — 장기 '신념' 레이어

| 항목 | 내용 |
|------|------|
| **역할** | 쌓인 경험 끝에 만든 깊은 신념 ("나는 자원 공유주의자다") |
| **차원** | 384차원 (sentence-transformers) |
| **업데이트** | t≥200 또는 메모리 100개 넘을 때만 (**조건부**, 비용 최소) |
| **emergent** | 같은 세계관끼리 클러스터 → 이념 집단 형성 → t~1000에 "혁명" 이벤트 자동 발생 |

**예시**: 초기 "생존"만 생각 → 5년 후 "자유 vs 통제" 세계관으로 갈라짐 → 한쪽은 협력 대형 집단, 다른 쪽은 고립 사멸. 3D 화면에서 색상별 이념 지도.

## 7. 입출력 흐름

```
[입력 레이어]
  뉴스·정책·데이터 → 영양분 형태로 4D 세계에 공급

[4D 세계 생성기]
  LangGraph + 4D 좌표 엔진 (완전 신규 개발)

[유기체 에이전트 풀]
  각 세포 = (x, y, z, t, 에너지, 유전자, 메모리, emotion, thought, worldview)

[시간 흐름 엔진]
  매 t: 성장/분열/사멸/융합/돌연변이 + Emotion 갱신
  매 10~50 t: Thought LLM
  매 200 t 또는 메모리 100+: Worldview LLM

[God View & 출력]
  3D 시각화 (Three.js) + 시간 슬라이더
  실시간 변수 주입 (예: t=500에 정책 변경)
  시나리오 인사이트 리포트 (의사결정 지원용)
```

---

## 8. God View

- **3D + 시간 슬라이더**: t를 움직이면 해당 시점의 4D 세계 상태를 3D로 시각화
- **Emotion 시각화**: 세포 색상(빨강=분노, 파랑=안정) + 크기(감정 강도) — 정책 주입 시 즉시 반응
- **실시간 주입**: t=500 같은 특정 시점에 정책·데이터를 주입 → 생태계 반응 즉시 관찰
- **What-if 시나리오**: "이 법이 통과되면?" 같은 질문에 대해 시뮬레이션 가능

## 10. 고민·연구 과제 (Difficult Parts)

### 10.1 고민해야 할 부분

| 이슈 | 내용 | 방향 |
|------|------|------|
| **규모 제어** | t 증가 시 세포 수 폭증 → 연산·메모리 한계 | 분열 cap, 포화 한계, 공간 샘플링 등 설계 |
| **LLM 호출 전략** | Thought(10~50t), Worldview(200t+) — 계층별 주기 분리 | Emotion은 규칙 기반 (LLM 0) |
| **t 단위 정의** | 1 step = 1일/1주/1년? | 시나리오 목적에 따른 캘린더 매핑 |
| **4D 거리 함수** | 융합·상호작용에 사용할 (x,y,z,t) 거리 | 공간 vs 시간 가중치, 정규화 방식 |

### 10.2 연구해야 할 부분

| 영역 | 질문 | 목적 |
|------|------|------|
| **Emergent 탄생** | 어떤 파라미터·초기 조건에서 의미 있는 패턴이 나오는가? | 반복 실험·튜닝 가이드 |
| **3계층 생성 품질** | Thought·Worldview LLM 출력의 일관성·해석 가능성 | 전략·신념 추출 검증 |
| **검증 방법** | 시뮬레이션 결과를 어떻게 평가·검증할 것인가? | "정확도" 대신 "해석 가능성·유용성" 기준 |
| **차별화** | 4D+5대 규칙+3계층 자생이 기존 ABM·LLM 에이전트 대비 어떤 우위를 갖는가? | 제품 포지셔닝 정교화 |

### 10.3 설계 체크리스트

- [ ] 규모 제어: 분열 cap, 포화 한계
- [ ] LLM 호출 전략: Thought 10~50t, Worldview 200t+ 조건부
- [ ] t 단위 정의: 1 step 의미, 총 step 수
- [ ] 4D 거리 함수: 융합·상호작용 조건
